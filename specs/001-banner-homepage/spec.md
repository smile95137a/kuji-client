## 使用者情境與測試

### 使用者故事 1 — 進入首頁看到輪播橫幅（優先級：P1）

玩家打開 KUJI 前台首頁，即可在頂部看到多張橫幅圖片自動輪播，點擊後導向對應店家頁面。

**此優先級的原因**：橫幅是品牌曝光與流量引導的核心入口，影響第一印象與轉換率。

**獨立測試**：
1. 確認後端有至少一筆 `status=PUBLISHED` 且在有效排程內的橫幅資料。
2. 打開 `/`，確認頂部輪播顯示橫幅圖片。

**驗收情境**：
1. **在** 首頁載入完成，**當** 頁面有已發布橫幅，**則** 輪播依 `displayOrder` 升序顯示橫幅，每張間隔 4 秒自動切換。
2. **在** 輪播播放中，**當** 玩家點擊右箭頭，**則** 立即切換到下一張橫幅並重置計時器。
3. **在** 首頁載入完成，**當** 後端返回空陣列（無已發布橫幅），**則** 輪播區域隱藏或顯示預設佔位圖，不顯示錯誤訊息。
4. **在** 輪播顯示，**當** 玩家點擊橫幅圖片，**則** 導向 `/stores/{storeId}` 店家頁。
5. **在** 僅剩一張橫幅，**當** 頁面顯示，**則** 左右箭頭隱藏，不顯示輪播指示點，圖片靜態展示。

### 使用者故事 2 — 行動裝置瀏覽橫幅（優先級：P2）

玩家使用手機瀏覽首頁，可透過左右滑動手勢切換橫幅。

**此優先級的原因**：行動裝置流量佔大多數，觸控操作體驗影響留存率。

**獨立測試**：
1. 使用 DevTools 模擬手機裝置，進入 `/`。
2. 滑動橫幅區確認手勢切換有效。

**驗收情境**：
1. **在** 行動裝置瀏覽首頁，**當** 玩家向左滑動橫幅，**則** 切換到下一張橫幅。
2. **在** 行動裝置瀏覽首頁，**當** 玩家向右滑動橫幅，**則** 切換到上一張橫幅。
3. **在** 行動裝置，**當** 橫幅顯示，**則** 圖片比例為 16:9 或 2:1，不變形、不裁切重要內容。

### 使用者故事 3 — 橫幅店家已停用（優先級：P2）

若後端返回的橫幅所屬店家已停用，前端不應顯示該橫幅。

**此優先級的原因**：後端 API 應只返回有效橫幅，但前端需優雅處理資料異常。

**獨立測試**：模擬 API 返回店家已停用的橫幅資料，確認前端不崩潰。

**驗收情境**：
1. **在** API 返回空圖片 URL 的橫幅，**當** 輪播嘗試渲染，**則** 跳過該項目，不顯示破圖。
2. **在** 所有橫幅過濾後為空，**當** 頁面渲染，**則** 顯示預設佔位圖。

## API 串接規格

### 橫幅資料

| 動作 | Method | 端點 | 主要 Request 欄位 | 主要 Response 欄位 | 前端 UI 狀態 |
|------|--------|------|------------------|--------------------|-------------|
| 取得已發布橫幅列表 | GET | `/api/banners` | 無 | `id`, `storeId`, `storeName`, `imageUrl`, `displayOrder`, `publishTime`, `unpublishTime` | 載入中 → 顯示輪播 / 顯示佔位圖 |

#### Request 範例
```
GET /api/banners
Authorization: （不需要，公開端點）
```

#### Response 範例
```json
[
  {
    "id": "uuid-1",
    "storeId": "store-uuid-1",
    "storeName": "扭蛋王國",
    "imageUrl": "https://cdn.example.com/banners/banner1.jpg",
    "displayOrder": 1,
    "publishTime": "2026-03-01T00:00:00",
    "unpublishTime": "2026-12-31T23:59:59"
  },
  {
    "id": "uuid-2",
    "storeId": "store-uuid-2",
    "storeName": "一番賞天堂",
    "imageUrl": "https://cdn.example.com/banners/banner2.jpg",
    "displayOrder": 2,
    "publishTime": "2026-03-01T00:00:00",
    "unpublishTime": null
  }
]
```

## 功能需求（前端 UI）

### 介面需求

- **FR-UI-001**：橫幅輪播元件應使用業界標準 Swiper/Carousel 函式庫（如 Swiper.js 或 Embla），支援觸控滑動、鍵盤導航及無障礙 ARIA 屬性。
- **FR-UI-002**：橫幅圖片需使用 `object-fit: cover` 保持比例，容器高度在桌機為 420px、平板為 300px、手機為 180px（或依設計稿調整）。
- **FR-UI-003**：輪播指示點（Dots）數量與橫幅數量一致，當前橫幅對應的圓點以不同顏色或尺寸突顯。
- **FR-UI-004**：左右箭頭在桌機顯示，在手機上隱藏（改用滑動手勢）。
- **FR-UI-005**：整個橫幅區塊為可點擊的 `<a>` 或 `<button>` 元素，導向 `/stores/{storeId}`。
- **FR-UI-006**：當 `imageUrl` 為空或圖片載入失敗時，顯示預設佔位圖（KUJI 品牌圖）。
- **FR-UI-007**：橫幅區塊下方接分類快捷入口（一番賞、扭蛋、刮刮樂、卡牌）以及熱門商品列表。
- **FR-UI-008**：頁面標題為「KUJI — 線上扭蛋・一番賞平台」，供 SEO 使用。

### 狀態管理

- **SM-001**：`bannersLoading` — API 請求進行中，顯示骨架屏（Skeleton）或 Spinner，橫幅高度保留。
- **SM-002**：`bannersError` — API 請求失敗，靜默失敗，輪播區域隱藏，不顯示錯誤訊息給用戶（非核心功能）。
- **SM-003**：`bannersEmpty` — API 返回空陣列，隱藏輪播，可顯示預設品牌橫幅圖（靜態資源）。
- **SM-004**：`currentIndex` — 當前輪播索引，初始值為 `0`。
- **SM-005**：`autoPlayTimer` — 自動播放計時器 ID，在使用者手動切換後重置，組件卸載時清除以防記憶體洩漏。

### 資料處理

- **DP-001**：API 回傳資料後，依 `displayOrder` 升序排列橫幅陣列（後端應已排序，前端雙重保險）。
- **DP-002**：過濾掉 `imageUrl` 為空或 `null` 的橫幅項目。
- **DP-003**：若 `storeId` 為空，禁用點擊（不導向），僅做裝飾展示。

## API 驗證清單

> 驗證後端 spec 與前端需求的一致性。

- **AV-001** ✅：`GET /api/banners` 回傳陣列格式，前端直接渲染 — 已確認一致。
- **AV-002** ✅：Response 包含 `storeId` 供前端生成導向連結 — 已確認。
- **AV-003** ✅：Response 包含 `displayOrder` 供前端排序 — 已確認。
- **AV-004** ✅：後端只返回 `status=PUBLISHED` 且在有效排程內的橫幅 — 後端已處理，前端無需自行過濾排程時間。
- **AV-005** ⚠️：`unpublishTime` 為 `null` 表示永久有效，前端無需客戶端過期檢查 — 需確認後端是否在 API 層面已過濾過期橫幅。
- **AV-006** ⚠️：`storeName` 是否一定存在於 Response？若店家已被刪除，`storeName` 可能為 `null` — 前端需做 null 保護。
- **AV-007** ❌：Response 未包含 `altText`（圖片替代文字）欄位 — 影響無障礙（a11y）與 SEO，建議後端補充或前端用 `storeName` 作為替代。
- **AV-008** ⚠️：API 是否有快取機制？建議前端設定 `stale-while-revalidate` 快取策略，避免每次進首頁都重新請求。
- **AV-009** ❌：Response 未包含橫幅的 `linkType`（是否為外部連結）欄位 — 現在固定導向店家頁，若未來需支援外部連結需後端擴充。

凡例：✅ 已確認一致 | ⚠️ 需要確認 | ❌ 發現缺口

## 成功標準

- **SC-001**：首頁在 3G 網路下，橫幅輪播的 LCP（Largest Contentful Paint）≤ 2.5 秒。
- **SC-002**：橫幅圖片使用 `loading="eager"` 預載第一張，其餘使用 `loading="lazy"`。
- **SC-003**：輪播自動播放間隔為 4 秒（±0.2秒），使用者互動後重置計時器。
- **SC-004**：點擊橫幅後，在 200ms 內觸發頁面導向。
- **SC-005**：在無橫幅情況下，首頁不顯示任何空白區塊或錯誤訊息，體驗完整。
- **SC-006**：輪播元件通過 WCAG 2.1 AA 標準的鍵盤導航測試（Tab 鍵可聚焦，Enter/Space 可點擊）。
